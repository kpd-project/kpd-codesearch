# KPD CodeSearch Web UI

---

## Проект
Веб-интерфейс для поиска по коду с использованием RAG

## Стек технологий
- **Фреймворк:** React + Vite (каталог `ui/`)
- **Стили:** Tailwind CSS
- **Компоненты:** shadcn/ui
- **Backend:** FastAPI (`web/`), WebSocket `/ws/state`

## Описание
Минималистичный тёмный веб-интерфейс для локального использования:
1. **Дашборд** — статистика репозиториев, количество чанков, статус Qdrant
2. **Панель управления** — добавить/удалить/переиндексировать репозитории
3. **Чат** — задавать вопросы о коде, получать ответы с подсветкой синтаксиса

## Язык
Русский — весь интерфейс на русском языке

## Стиль
Минималистичный чёрно-белый дизайн
- Фон: #000000 (чёрный)
- Карточки: #111111 (почти чёрный)
- Границы: #333333 (тёмно-серый)
- Основной цвет: #ffffff (белый)
- Текст: #ffffff (белый)
- Вторичный текст: #888888 (серый)
- Без цветных акцентов — только чёрное и белое

## Разделы страницы
1. **Хедер:** Название "KPD CodeSearch", индикатор статуса (точка), кнопка настроек
2. **Сайдбар:** Список репозиториев с чекбоксами и кнопками действий
3. **Основная область:** Вкладки "Чат" и "Панель управления"
4. **Футер:** Версия приложения, статус подключения к Qdrant

## Функционал
- Управление репозиториями (просмотр списка, добавление, удаление, переиндексация с прогрессом)
- Чат с потоковыми ответами от ИИ
- Подсветка синтаксиса кода в сообщениях
- Real-time обновления через WebSocket (прогресс индексации)
- Настройки параметров модели (выбор LLM, temperature, top-k)
- Индикатор статуса подключения к Qdrant

## Компоненты
- ИндикаторСтатуса (белая/серая точка)
- КарточкаРепозитория (название, чекбокс, кнопки "Переиндексировать" и "Удалить", прогресс-бар)
- ИнтерфейсЧата (список сообщений, поле ввода, кнопка отправки)
- МодальноеОкноНастроек (выпадающие списки, ползунки)
- Вкладки (Чат | Панель управления)
- ПрогрессБар

## API эндпоинты (пока мокапы)
- GET /api/repos — список репозиториев
- POST /api/repos — добавить репозиторий
- DELETE /api/repos/:name — удалить репозиторий
- POST /api/repos/:name/reindex — переиндексировать
- POST /api/query — отправить вопрос в чате
- GET /api/status — статус системы
- WebSocket /ws/state — real-time обновления
